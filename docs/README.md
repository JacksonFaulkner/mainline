# Docs Guide

This folder contains implementation notes, migration plans, and visual assets for the project.

## What Is Here

- `plan.md`: high-level migration and delivery plan.
- `backend_migration.md`: backend migration details and sequencing.
- `frontend_migration.md`: frontend migration details and sequencing.
- `optimizations.md`: performance and system-level tuning notes.
- `fast_api_template.md`: captured upstream template reference notes.
- `img/`: markdown assets used by docs pages.

## How To Use This Folder

- Treat these files as project context and architectural history.
- Keep runnable source-of-truth in `backend/` and `frontend/`.
- Update docs when behavior, routes, or deployment assumptions change.
